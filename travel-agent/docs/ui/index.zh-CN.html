设计关键词

简约、自然、可信赖、适合旅游规划与对话式助手。

输出方式

全部为原生 HTML 页面结构，后续可直接迁移到 Vue / React / Next.js。

主要场景

游客咨询与行程规划、运营上传资料与维护 RAG 知识库、移动端随时追问和查看日程。

视觉方向

采用淡青绿和晴空蓝作为辅助色，减少厚重阴影，强化留白和圆角，避免此前图层堆叠带来的拥挤和重叠问题。

交互策略

用户端突出对话输入、推荐灵感、预算与天气摘要；后台突出上传、更新、分段、启用状态和召回质量反馈。

适配方式

桌面端采用结构化双栏布局，移动端采用单列卡片与底部输入区，保证在手机上也能快速追问和查看路线。
